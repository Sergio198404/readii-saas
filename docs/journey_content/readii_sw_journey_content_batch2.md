### Phase 1：现有员工合规审计

**Phase 说明**：雇员入职第一天需立即完成，确保所有初始合规信息完整记录。这是英国雇主的法定义务，也是 Sponsor Licence 审核时 UKVI 最优先核查的内容。

---

#### 1.1 雇佣合同签署并留档

**合规依据**：Employment Rights Act 1996 s.1；雇员有权在入职第一天收到书面工作条款声明（Statement of Particulars）。

**必须包含的内容**：
- 雇主和雇员全名及地址
- 工作开始日期
- 职位名称和工作内容描述（**注意：职位描述须与 CoS 申请中的描述完全一致**）
- 工作地点
- 薪资（≥£41,700/年 或所在 SOC Code 的 going rate，取高值）及支付频率
- 工作时间（标准 35-40 小时/周，需与 Sponsor Licence 申请的 going rate 测算一致）
- 年假权益（法定 28 天，含法定节假日）
- 试用期（如有，通常 3-6 个月）
- 终止通知期（法定最低：雇员首月不设 / 之后每年增加 1 周，最多 12 周）
- 养老金计划信息（Auto-Enrolment 相关）
- 适用的纪律和申诉程序

**Readii 提供的模板**：《Readii 标准雇佣合同模板 v1.0》（含 Sponsor Licence 合规条款）

**证明文件**：双方签字的雇佣合同扫描件（PDF）

**常见错误**：
- 职位描述与 CoS 不一致 → UKVI 可能质疑雇佣真实性
- 薪资低于 going rate → 直接导致工签被拒
- 没有中文版本 → 雇员可能对合同条款不清楚（建议同时准备中英双语版）

---

#### 1.2 Right to Work 核查完成并留档

**合规依据**：Immigration Act 2014；雇主有法定义务在雇员入职前核查其在英国工作的权利，否则面临最高 £45,000 的罚款。

**核查步骤**：
1. 要求雇员出示 Graduate Visa 或其他合法签证的原件（护照 + Vignette 贴纸 或 BRP 卡）
2. 在雇员本人在场的情况下，核验证件真实性（有效期、照片匹配）
3. **拍摄清晰彩色照片存档**（护照信息页 + 签证页，或 BRP 两面）
4. 在记录表上注明：核查日期、核查人姓名、核查类型（原件核查）
5. 如签证为电子签（eVisa），通过 UKVI 在线核查服务（Home Office Employer Checking Service）核验并截图存档

**存档期限**：雇佣关系结束后至少 2 年内保留

**注意事项**：
- 每次签证续签后须重新核查（如果到期日期变更）
- 不能只看复印件，必须核查原件
- 核查记录必须包含日期，不能补填

**证明文件**：Right to Work 核查记录表（含日期、核查人、证件信息）+ 护照/签证/BRP 照片

**Readii 提供的模板**：《Right to Work 核查记录表》

---

#### 1.3 P45/P46 新员工信息存档

**合规依据**：Income Tax (PAYE) Regulations 2003

**P45（来自上一份工作）**：
- 如果雇员有上一份英国工作，请雇员提供 P45
- P45 包含：上一份工作的离职日期、全年累计收入、已缴税款
- 将 P45 信息转交会计事务所，用于设置正确的 Tax Code

**P46（无 P45 的情况）**：
- 雇员是第一次在英国工作，或没有前雇主 P45
- 填写 P46（新员工信息表），包含：全名、NI Number、地址、是否为主要工作
- 同样转交会计事务所

**注意事项**：
- 没有 NI Number 的新雇员需申请 NI Number（可在入职后申请，但工资申报前最好已获取）
- 如无 P45 且 NI Number 未知，会计所会使用 Emergency Tax Code，可能导致雇员多缴税

**证明文件**：P45 扫描件 或 P46 填写记录

---

#### 1.4 Tax Code 和 NI Number 记录

**合规依据**：HMRC 的正确 Tax Code 设置是雇主义务；NI Number 是工资申报必要信息。

**Tax Code**：
- 通常会计事务所根据 P45 或 P46 设置
- 标准 Tax Code 2025-26 年度为 1257L（个人免税额 £12,570）
- 如有特殊情况（如兼职、养老金收入等），Tax Code 会有所不同
- 将 HMRC 设置的 Tax Code 记录在员工档案中

**NI Number**：
- 英国国家保险号码，格式：两个字母 + 六个数字 + 一个字母（如 AB123456C）
- 在英国长期居住的人通常已有 NI Number
- 新来英国的境外雇员（路径 B 入境后）需申请 NI Number
- NI Number 申请：通过 HMRC 在线申请或致电 HMRC 申请

**证明文件**：Tax Code 通知记录、NI Number 确认（可来自 P45 或 HMRC 信函）

---

#### 1.5 Pension Auto-Enrollment 状态确认

**合规依据**：Pensions Act 2008；所有符合条件的雇员必须被自动加入工作场所养老金，否则雇主面临 TPR（The Pensions Regulator）处罚。

**自动加入条件**（同时满足以下三项）：
- 年龄在 22 岁至国家养老金年龄之间
- 在英国工作
- 年收入超过 £10,000（2025-26 年度）

**雇主义务**：
1. 在雇员入职后 **6 周内** 完成 Auto-Enrollment（如果符合条件）
2. 选择一家经 TPR 认可的养老金计划（推荐：NEST - National Employment Savings Trust，政府推荐的免费方案）
3. 雇主必须缴纳 ≥3% 的雇主供款（基于合格收入）
4. 雇员缴纳 ≥5%（含税收减免后为 4%）

**Opt-Out 处理**：
- 符合条件的雇员可以选择退出（Opt-Out），但不能被强制要求退出
- 如雇员选择退出，必须提交书面 Opt-Out 通知，保留记录

**证明文件**：养老金加入确认书（来自养老金提供商）或退出通知（如适用）

---

#### 1.6 Holiday Entitlement 计算正确

**合规依据**：Working Time Regulations 1998

**法定年假标准**：
- 全职雇员：28 天（5.6 周），含 8 个法定节假日（England & Wales）
- 或：20 天年假 + 8 个法定节假日分开计算
- 如合同给予更多年假（如 25 天 + 8 天），以合同为准

**计算注意事项**：
- 试用期内的假期按比例累积（不能规定试用期内无年假）
- 年假不得用金钱代替（在职期间）；离职时未使用假期须结算
- 兼职雇员按工作天数比例计算

**需要核实**：
- 合同中年假条款是否至少等于法定最低标准 28 天
- 年假制度（是否包含法定节假日、年假年度（4月1日至3月31日 或 自然年等））
- 假期管理系统（建议使用简单的假期追踪表）

**证明文件**：合同中假期条款 + 年假管理记录表（Readii 提供模板）

---

#### 1.7 Working Time Opt-Out（超 48 小时/周）

**合规依据**：Working Time Regulations 1998 Regulation 4；欧盟工时指令

**基础规定**：
- 英国工人每周工作时间不得超过平均 48 小时（以 17 周为滚动平均计算期）
- 这是法定保护，雇主不得强制雇员超时工作

**Opt-Out 协议**：
- 如果业务需要雇员经常超过 48 小时/周工作，雇员可以自愿签署 Opt-Out 协议
- Opt-Out 是完全自愿的，雇员可以随时撤回（给予 7 天书面通知）
- 不能将签署 Opt-Out 作为入职条件

**适用场景**：
- 初创公司早期阶段，创始人级雇员经常超时工作
- 建议：如有需要，在入职时提供 Opt-Out 选项，但不强制

**证明文件**：已签署的 Working Time Opt-Out 协议（如适用）或"不适用"的确认记录

---

#### 1.8 Data Protection Notice 已发放

**合规依据**：UK GDPR Article 13/14；雇主作为员工数据的 Data Controller，必须在收集员工数据时告知相关信息。

**必须告知的内容（雇主须提供）**：
- 公司（Data Controller）的名称和联系方式
- 处理员工数据的目的和法律依据（合同履行、法定义务等）
- 数据保留期限
- 员工的数据权利（查阅权、纠错权、删除权等）
- 是否向第三方传输数据（如会计事务所、HMRC）

**发放方式**：
- 作为独立文件随雇佣合同一同发放
- 或在雇佣合同中设专门章节说明
- 雇员需签字确认已收到并阅读

**证明文件**：雇员签字确认的《员工数据保护通知》（Readii 提供模板）

### Phase 2：雇佣合同和政策文件

**Phase 说明**：建立公司的核心 HR 政策体系，确保有书面政策应对常见劳动关系情况，避免劳动争议。这些文件也是 UKVI 合规检查时可能要求核验的"公司真实运营"证明之一。

---

#### 2.1 雇佣合同（完整版）

详见 1.1；此项确认 1.1 的完整版合同已完成，包含所有法定条款及公司特定条款。

**额外核查**：
- 合同是否包含竞业禁止条款（如有，确认范围合理且可执行）
- 是否包含知识产权归属条款（雇员在职期间创造的 IP 归公司所有）
- 保密协议（NDA）是否已独立签署

**证明文件**：最终完整版雇佣合同（含所有附件）

---

#### 2.2 Disciplinary and Grievance Policy

**合规依据**：Employment Act 2002；ACAS Code of Practice on Disciplinary and Grievance Procedures

**必须包含的内容**：

纪律程序（Disciplinary Procedure）：
- 什么行为构成违规（misconduct）
- 什么行为构成严重违规（gross misconduct）—— 可能导致即时解雇
- 调查程序
- 书面警告、最终书面警告的程序
- 解雇决定的程序
- 上诉权利

申诉程序（Grievance Procedure）：
- 员工如何正式提出申诉
- 初步讨论（非正式解决）
- 书面申诉的处理流程
- 申诉结果的通知方式
- 上诉权利

**注意事项**：
- ACAS Code of Practice 在劳动争议仲裁时具有法律参考地位
- 公司规模较小（1-5 人）时，可使用简化版政策

**证明文件**：员工签字确认的《纪律与申诉政策》（Readii 提供模板）

---

#### 2.3 Health & Safety Policy

**合规依据**：Health and Safety at Work Act 1974 s.2(3)；超过 5 名员工必须有书面 H&S 政策

**基本内容**（初创小公司简化版）：
- 雇主的 Health & Safety 总体承诺声明
- 具体职责分工（谁负责 H&S 管理）
- 主要风险评估结果（办公室环境通常是低风险）
- 火灾疏散程序和紧急联系方式
- 报告事故/险情的程序
- 员工的 H&S 义务

**DSE 风险评估**（Display Screen Equipment）：
- 长期使用电脑的雇员须完成 DSE 评估（详见 4.7）
- H&S 政策应包含 DSE 管理的条款

**证明文件**：书面《Health & Safety Policy》（雇主签字）

---

#### 2.4 Code of Conduct

**内容**：公司对员工行为的基本期望，涵盖：

- 职业行为标准（着装、出勤、沟通礼仪）
- 保密义务（客户信息、公司信息不得外泄）
- 利益冲突声明（须主动申报）
- 社交媒体使用规范（工作时间、公司相关内容发布）
- 电子设备使用政策（公司设备不得用于个人目的）
- 骚扰和歧视的零容忍政策
- 举报机制（Whistleblowing）

**注意事项**：
- 本文件与合同中的某些条款重叠，但作为独立文件更便于更新
- 雇员须签字确认已阅读并同意遵守

**证明文件**：员工签字的《员工行为准则》

#### 3.2 Payroll 系统运行正常

**合规依据**：Income Tax (PAYE) Regulations 2003；Real Time Information (RTI) 申报要求

**RTI 申报要求**：
- 每次发工资时（或在发工资当日），雇主须向 HMRC 提交 Full Payment Submission（FPS）
- FPS 须在支付薪资的**当天或之前**提交（最晚不超过付薪日）
- 延迟提交 FPS 会产生罚款（£100-400/月，按员工人数）

**首次 RTI 申报确认**：
- 确认会计事务所已提交第一份 FPS
- 确认 HMRC 已收到（可在 Business Tax Account 查询）
- 确认薪资已按时发入雇员账户

**月度工资单（Payslip）**：
- 每次发薪须提供书面或电子工资单
- 工资单须包含：工资总额（Gross Pay）、各项扣除（NI、Income Tax、养老金）、实发金额（Net Pay）

**证明文件**：首份 RTI 申报提交确认（HMRC Business Tax Account 截图）

---

#### 3.3 Pension Scheme 和通知信已发放

**合规依据**：Pensions Act 2008；The Pensions Regulator（TPR）

**两类通知信必须发放**：

1. **Auto-Enrolment 通知信**（符合条件的雇员）：
   - 必须在评估雇员资格后 6 周内发出
   - 说明雇员已被自动加入养老金计划
   - 告知雇主和雇员的供款比例
   - 说明退出（Opt-Out）的权利和方式

2. **不符合条件的通知信**（若雇员暂不符合 Auto-Enrolment 条件）：
   - 即使暂时不符合，也须给雇员发送"你目前不在 Auto-Enrolment 范围内"的通知

**TPR 登记义务**：
- 首次完成 Auto-Enrolment 后须向 TPR 声明合规（Declaration of Compliance）
- 通常须在 Auto-Enrolment 完成后 5 个月内提交

**证明文件**：养老金加入通知信副本 + TPR 声明合规确认

---

#### 3.4 Data Protection Policy（公司级）

**合规依据**：UK GDPR；Data Protection Act 2018

**与 1.8 的区别**：
- 1.8（员工 Data Protection Notice）是告知员工公司如何处理**员工数据**
- 3.4（公司级 Data Protection Policy）是关于公司整体如何处理所有数据（包括客户数据）

**公司级 Data Protection Policy 必须覆盖**：
- 公司收集哪些类型的个人数据（员工数据、客户数据等）
- 数据收集的法律依据
- 数据安全保护措施
- 数据保留和删除政策
- 数据泄露响应程序（72 小时内须向 ICO 报告）
- 员工数据访问权限管理

**证明文件**：书面《公司数据保护政策》（董事签字）

---

#### 3.5 ICO Registration 完成

**合规依据**：Data Protection Act 2018 s.108；几乎所有处理个人数据的组织必须向 ICO 注册。

**判断是否需要注册**：
- 几乎所有商业公司都需要注册（包括保存员工信息、客户联系方式）
- 少数豁免情况：纯为个人或家庭目的处理数据（不适用于商业公司）

**注册方式**：
- 在 ico.org.uk/registration 在线注册
- 年费：Tier 1（最多 10 员工，营业额 ≤ £632K）£40/年；Tier 2（最多 250 员工）£60/年

**注册内容**：
- 公司名称和注册地址
- 处理的数据类型（员工数据、客户数据等）
- 数据处理的目的

**完成后**：ICO 颁发注册证书（含注册号），必须保留

**证明文件**：ICO 注册证书（含注册号）

---

#### 3.6 相关 HMRC 注册完成

**核查清单**：

| 注册项目 | 说明 | 完成证明 |
|---|---|---|
| Corporation Tax（UTR） | 公司注册后 3 个月内完成，获取 10 位 UTR | HMRC 发出的 UTR 通知信 |
| PAYE Scheme | 雇员入职前必须激活 | PAYE 参考号确认 |
| VAT 注册（如适用） | 年营业额 ≥£90,000 须注册；可提前自愿注册 | VAT 注册证书（含 VAT 号） |
| Construction Industry Scheme（CIS，如适用） | 建筑行业公司使用分包商时须注册 | CIS 注册确认 |

**注意事项**：
- UTR 通知信通常在公司注册后 4-8 周内邮寄至注册地址
- 如果公司地址是虚拟地址，须确保信件被正常转发
- PAYE 参考号格式：XXX/XXXXXXX（3位数字/7位数字）

**证明文件**：所有适用项目的 HMRC 注册确认文件集合

#### 4.2 Recruitment Policy & Process

**内容**：公司正式的招聘流程文件，须包含：

- 职位空缺的内部审批流程（谁批准开新职位）
- 职位描述（Job Description）的制定程序（**重要：与 CoS 申请的 JD 必须一致**）
- 招聘渠道（内部优先、外部招聘网站、猎头等）
- 简历筛选标准和面试流程
- 录用决定程序（谁有权发出 Offer）
- Offer Letter 内容要求
- 背景调查触发条件

**Sponsor Licence 特别要求**：
- 公司需证明在招聘 sponsored worker 之前，已按 Resident Labour Market Test 精神进行过市场招聘（虽然正式的 RLMT 已取消，但 UKVI 仍会评估岗位的 genuine 需求）
- 建议记录招聘过程（即使是家族相关安排，也需要有"为什么这个人适合"的文件记录）

**证明文件**：书面《招聘政策》文件

---

#### 4.3 Equal Opportunities Policy

**合规依据**：Equality Act 2010；禁止在就业中基于保护特征（Protected Characteristics）进行歧视

**9 项受保护特征**：
- 年龄
- 残疾
- 性别重置
- 婚姻和民事伴侣关系（仅工作领域）
- 怀孕和育儿
- 种族（含国籍和民族）
- 宗教或信仰
- 性别
- 性取向

**政策内容**：
- 公司对平等就业机会的承诺声明
- 适用范围（招聘、培训、晋升、薪资等各方面）
- 举报歧视的程序
- 对违反政策行为的纪律处理

**注意事项**：
- 不能在招聘广告中包含任何可能被视为歧视的语言
- 面试问题不能涉及保护特征（如不能问"你计划要孩子吗？"）

**证明文件**：书面《平等就业机会政策》

---

#### 4.4 Background Checks Process

**内容**：公司进行背景调查的标准流程：

**标准背景调查内容**（可根据岗位调整）：
- 身份核验（已由 Right to Work 核查涵盖）
- 就业历史核实（Reference Check，向前雇主核实）
- 学历证书核验（如岗位有学历要求）
- 专业资质核验（如有 RQF Level 6 要求）
- DBS 核查（Disclosure and Barring Service）：对于特定岗位（教育、医疗、社会服务）必须

**DBS 核查说明**：
- 基础 DBS：无前科记录核查
- 标准 DBS：已花费 / 未花费的定罪记录
- 增强 DBS：用于涉及儿童或弱势群体的岗位
- 对于 IT/商务咨询等一般岗位，DBS 不是强制要求

**流程文件须包含**：
- 哪些岗位需要何种背景调查
- 何时进行（通常在有条件 Offer 之后）
- 候选人不通过背景调查时的处理方式
- 数据保护（背景调查结果的存储和删除）

**证明文件**：书面《背景调查流程》文件

---

#### 4.5 Right to Work Checks（入职前专项核查）

本项与 1.2 配合，确认入职**前**（不是入职后）的 Right to Work 核查已成为标准流程。

**关键时序**：
- 核查必须在雇员**实际开始工作前**完成
- 不能先让员工工作，事后再核查
- 如果发现右工权利有限（如签证将在 3 个月内到期），须制定后续处理计划

**连续 Right to Work 监控**：
- 建立提醒系统，在签证到期前 2-3 个月主动联系雇员确认续签状态
- 如签证未续签，雇主须停止雇佣（否则构成违法雇佣）

**证明文件**：入职前 Right to Work 核查记录（格式同 1.2）

---

#### 4.6 Payroll Starter Form（新员工首次工资申报表）

**HMRC Starter Checklist（替代旧版 P46）**：

新员工须在首次工资申报前完成 HMRC Starter Checklist，包含：
- 个人信息（全名、出生日期、地址）
- NI Number
- 是否为主要工作（Statement A/B/C）：
  - Statement A：这是第一份工作 / 失业后第一份工作（免税额全额适用）
  - Statement B：有其他工作或养老金（按基础税率扣税）
  - Statement C：有其他工作且其他工作已使用全部免税额（按高税率扣税）
- 学生贷款还款状态（如有）
- 养老金信息（如已加入其他养老金计划）

**操作**：
- 雇员填写后，HR/会计事务所据此设置正确的 Tax Code
- 原件存档于员工档案

**证明文件**：雇员签署的 HMRC Starter Checklist

---

#### 4.7 DSE Assessment（显示屏设备工作评估）

**合规依据**：Health and Safety (Display Screen Equipment) Regulations 1992

**适用范围**："Habitual User"——每天使用显示屏工作超过 1 小时的雇员

**评估内容**（雇主义务）：
- **显示屏**：亮度、对比度、字符清晰度、无频闪
- **键盘**：可调整角度，与显示器分开
- **鼠标/输入设备**：可正常使用，不造成手腕不适
- **桌子**：足够空间放置设备，有文件夹
- **椅子**：可调整高度，腰部支撑，稳定
- **工作环境**：照明充足无强光，噪音合理，温度适宜
- **软件**：运行稳定，界面友好
- **工作模式**：是否有合理的休息时间（每小时建议休息 5-10 分钟）

**雇主还须**：
- 提供视力检查（雇员要求时，雇主需提供 / 资助）
- 提供适当的培训和信息

**操作方式**：
- 雇员自评问卷（Readii 提供模板）
- 如发现问题，安排专业人员评估并改善

**证明文件**：完成的 DSE 自评问卷（雇员签字）

---

#### 4.8 Training 记录

**合规依据**：雇主有责任确保员工具备履行职责的必要培训和技能。

**必须记录的培训类型**：

| 培训类型 | 频率 | 备注 |
|---|---|---|
| 入职培训（Induction） | 入职第一周 | 公司概况、政策、安全、系统 |
| Health & Safety 基础培训 | 入职时 + 定期复训 | 法定要求 |
| 数据保护培训（GDPR） | 入职时 + 年度复训 | 处理个人数据的所有员工必须 |
| 岗位专业技能培训 | 持续 | 根据岗位要求 |
| 软件/系统使用培训 | 入职时 | 会计软件、邮件系统等 |

**记录格式**（每次培训须记录）：
- 培训名称和内容概述
- 培训日期和时长
- 培训形式（课堂/在线/实操）
- 参加人员
- 完成确认（培训者和受训者签字）

**与 Sponsor Licence 的关联**：
- UKVI 合规检查时可能要求查看员工培训记录
- 特别是 sponsored worker 的培训记录，需证明员工确实在学习和成长（genuine employment 证明之一）

**证明文件**：员工培训记录表（Readii 提供模板，含签到/完成确认）

### Appendix A 概述

Sponsor Licence 申请的 Appendix A 要求申请人提交**至少 4 份证明文件**，但实际上 Readii 会准备完整的 10+ 份文件包，以降低 UKVI 补件要求的风险，并增强申请的可信度。

UKVI 对文件的核心评估标准：
- **真实性**：文件是否来自可验证的官方来源
- **时效性**：文件是否在有效期内（银行对账单通常要求 6 个月内）
- **一致性**：所有文件中的公司名称、地址、CRN 是否完全一致
- **完整性**：文件是否完整（无缺页、无遮挡、清晰可读）

### 文件分类 A：公司身份证明

#### A-1 Companies House 注册证书（Certificate of Incorporation）

**来源**：Companies House 官网下载或纸质原件扫描
**内容要求**：显示公司全名、注册号（CRN）、注册日期、注册地点
**格式要求**：彩色扫描件，所有内容清晰可读
**有效期说明**：永久有效（公司存续期间）
**注意事项**：文件上的公司名称必须与申请表格中填写的完全一致（含有限公司标识"Limited"或"Ltd"）
**状态指示**：✓ 已获取 / ⚠ 待上传

**内容要求**：显示公司名称、CRN、注册地址、当前 Director 名单、股东名单及持股比例
**格式要求**：网页截图，须包含 Companies House 官网 URL 和截图日期

### 文件分类 B：经营证明（核心，权重最高）

#### B-1 银行对账单（近 6-12 个月，连续无间断）

**来源**：公司银行账户的正式对账单
**内容要求**：
- 显示公司全名（与注册证书一致）
- Sort Code + Account Number
- 月度起止日期
- 所有收入和支出明细（不能有空白月份）
- 银行标志和官方格式

**时效要求**：必须包含到申请日前 1 个月的记录（例如：5 月提交申请，对账单须覆盖至 4 月）
**月份数量**：至少 6 个月（推荐 12 个月）
**格式要求**：银行原始 PDF 格式，不接受截图

**高街银行 vs 数字银行的注意事项**：
- 高街银行（Barclays、Lloyds、HSBC、NatWest）：UKVI 接受度最高
- 数字银行（Revolut Business、Monzo Business）：UKVI 接受，但可能触发额外审查，建议同时附上高街银行拒绝记录作为补充说明

**⚠️ 关键提醒**：银行对账单上每一笔大额入账，必须有对应的发票和合同（与财务健康检查阶段 15 联动），确保"每笔收入有据可查"

---

#### B-2 销售合同/服务协议样本（2-5 份）

**来源**：公司与客户签署的实际合同
**内容要求**：
- 合同各方名称（显示公司名称）
- 合同日期（须在对账单收入覆盖期间内）
- 服务内容描述（须与 BP 描述的业务方向一致）
- 合同金额（须与对账单入账金额匹配）
- 各方签字

**脱敏处理**：如客户不希望公开详细信息，可以遮去客户名称，但合同金额和服务内容必须完整

**数量建议**：提交 2-5 份，涵盖不同客户和不同时间段，展示业务的持续性

---

#### B-3 发票记录（与银行对账单入账对应）

**来源**：公司对外开具的发票
**内容要求**：
- 发票头（含公司名称、CRN、VAT 号、注册地址）
- 发票号（连续编号）
- 开票日期
- 客户信息（可部分脱敏）
- 服务描述
- 金额（含 VAT 说明）
- 付款方式和账户信息

**核验要求**：发票金额 + 开票日期须能在银行对账单中找到对应的入账记录

## 三、Key Personnel 资质审核清单

> 对应 Journey 阶段 19。
> Sponsor Licence 获批后立即执行，作为 SMS 设置的前置步骤。

---

### 三个角色说明

| 角色 | 英文 | 职责 | 可以是同一人吗？ |
|---|---|---|---|
| AO | Authorising Officer | 负责所有 sponsored workers 的招聘和管理 | ✓ 三角色可由同一人担任 |
| Key Contact | Key Contact | UKVI 和公司的主要联系人 | ✓ |
| Level 1 User | Level 1 User | SMS 系统的主要操作员（发 CoS、更新信息等）| ✓ |

**重要新规（2024.12.31 起）**：
- Level 1 User 中**至少一人**必须是 UK settled worker
- UK settled = 英国国籍 / 英国永居 / EU Settlement Scheme Settled Status / Indefinite Leave to Remain
- Level 1 User 必须是公司的 employee、director 或 partner，不能是外部顾问

---

### 核查项目 1：永久在英居住和工作权利

**核查依据**：UKVI Sponsor Guidance Part 1 L14.4

**可接受的身份类型**：
- 英国国籍（British Citizenship）→ 护照
- 英国永久居留（ILR / Indefinite Leave to Remain）→ BRP 卡或内政部信函
- EU Settlement Scheme Settled Status → 在线核查
- 某些其他特定签证（如 Innovator Founder Visa 持有人可以担任 AO）

**核查步骤**：
1. 要求候选人提供身份证明文件
2. 核验文件真实性（有效期、照片匹配）
3. 如有疑问，通过 UKVI 在线核查服务验证
4. 拍照存档

**不接受的身份**：
- 仅持有时限性签证（如学生签证、工作签证、访客签证）

---

### 核查项目 2：没有犯罪记录

**核查依据**：UKVI Sponsor Guidance Part 1 L14.5

**核查方式**：
- 当前阶段：诚信声明（Self-Declaration）+ Readii 基于公开信息的初步查询
- 建议（非强制）：申请 DBS Basic Check（基础犯罪记录核查）

**DBS Basic Check 申请**：
- 可通过 gov.uk 官网在线申请
- 费用：£18
- 处理时间：通常 2-5 个工作日
- 结果：显示未排除的犯罪记录（或"无记录"）

**Readii 的操作**：
- 提供 Key Personnel 诚信声明模板（自我声明无犯罪记录）
- 建议申请 DBS Basic Check 作为书面证明
- 记录核查日期和结果

---

### 核查项目 3：没有破产限制令（Bankruptcy Restrictions Order）

**核查依据**：UKVI Sponsor Guidance Part 1 L14.6(e)

**核查方式**：
- 在 Insolvency Service 的 Individual Insolvency Register 上查询（免费公开查询）
- 网址：esi.insolvency.service.gov.uk/iri/esi/search.do
- 输入候选人姓名进行搜索

**如查询结果为"无记录"**：截图存档

---

### 核查项目 4：没有债务减免限制令（Debt Relief Restriction Order）

**核查方式**：同上，在 Individual Insolvency Register 中同时查询

---

### 核查项目 5：未被法律禁止担任公司董事

**核查方式**：
- 在 Companies House 的 Disqualified Directors Register 查询
- 网址：find-and-update.company-information.service.gov.uk/search/disqualified-officers
- 输入候选人姓名搜索

**如查询结果为"无记录"**：截图存档

---

### 核查项目 6：与被吊销 Sponsor Licence 公司的关联性检查

**核查内容**：候选人是否曾担任过以下公司的 Director：
- 曾被 UKVI 拒绝 Sponsor Licence 的公司
- 曾被吊销 Sponsor Licence 的公司

**核查方式**：
- 在 Companies House 上查询候选人的历史任职记录（Directors 标签下 → Current/Previous Appointments）
- 逐一核查历史任职公司是否在 UKVI 的 Register of Licensed Sponsors 中

**注意**：此项较难自动化，Readii 人工完成

---

### 核查项目 7：与公司的雇佣关系确认（Level 1 User 必须项）

**核查内容**：Level 1 User 是否为公司的 employee / director / partner

**接受的证明**：
- 在 Companies House 登记为 Director → 官网截图
- 雇佣合同 → 合同扫描件
- 薪资记录显示该人在公司薪资单上 → RTI 或工资单

## 四、数据库录入说明（Code 使用）

### HR 合规子模块表结构

```sql
-- HR 合规项目定义表（静态数据，不随客户变化）
CREATE TABLE IF NOT EXISTS hr_compliance_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  item_code TEXT UNIQUE NOT NULL, -- '1_1', '1_2', ... '4_8'
  phase_number INT NOT NULL CHECK (phase_number BETWEEN 1 AND 4),
  item_number TEXT NOT NULL, -- '1.1', '1.2'
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  compliance_basis TEXT, -- 法律依据
  template_url TEXT, -- Readii 模板下载链接
  evidence_type TEXT NOT NULL, -- 需要上传什么证明文件
  is_mandatory BOOLEAN DEFAULT true,
  applies_to TEXT DEFAULT 'all' CHECK (applies_to IN ('all', 'path_a', 'path_b', 'conditional')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 客户的 HR 合规进度（每个客户一套记录）
CREATE TABLE IF NOT EXISTS customer_hr_compliance (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  customer_id UUID NOT NULL REFERENCES customer_profiles(id) ON DELETE CASCADE,
  item_id UUID NOT NULL REFERENCES hr_compliance_items(id),
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'in_progress', 'completed', 'waived')),
  completed_by TEXT, -- 'customer' or 'readii'
  completed_at TIMESTAMPTZ,
  evidence_url TEXT, -- 上传的证明文件
  evidence_file_name TEXT,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(customer_id, item_id)
);

-- Appendix A 清单进度
CREATE TABLE IF NOT EXISTS customer_appendix_a (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  customer_id UUID NOT NULL REFERENCES customer_profiles(id) ON DELETE CASCADE,
  doc_code TEXT NOT NULL, -- 'A-1', 'A-2', 'B-1'...
  doc_name TEXT NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'uploaded', 'verified', 'rejected')),
  file_url TEXT,
  file_name TEXT,
  verified_by UUID REFERENCES profiles(id),
  verified_at TIMESTAMPTZ,
  rejection_reason TEXT,
  expiry_date DATE, -- 如有有效期（如保险、对账单）
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(customer_id, doc_code)
);

-- Key Personnel 审核记录
CREATE TABLE IF NOT EXISTS key_personnel_reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  customer_id UUID NOT NULL REFERENCES customer_profiles(id) ON DELETE CASCADE,
  person_name TEXT NOT NULL,
  roles TEXT[] NOT NULL, -- ['ao', 'key_contact', 'level1_user']
  uk_settled_status TEXT CHECK (uk_settled_status IN ('british_citizen', 'ilr', 'eu_settled', 'innovator_visa', 'other')),
  uk_settled_evidence_url TEXT,
  dbs_check_completed BOOLEAN DEFAULT false,
  dbs_check_date DATE,
  dbs_check_result TEXT CHECK (dbs_check_result IN ('clear', 'has_record', 'pending')),
  bankruptcy_check_clear BOOLEAN,
  bankruptcy_check_date DATE,
  debt_relief_check_clear BOOLEAN,
  director_disqualified_check_clear BOOLEAN,
  historic_sponsor_check_clear BOOLEAN,
  employment_verified BOOLEAN, -- Level 1 User 只需此项
  review_status TEXT DEFAULT 'pending' CHECK (review_status IN ('pending', 'passed', 'failed', 'action_required')),
  reviewed_by UUID REFERENCES profiles(id),
  reviewed_at TIMESTAMPTZ,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
